# 北航博雅课程 Agent — 业务与发展规划 (Roadmap)

> 整理项目当前已实现的业务逻辑，以及未来的演进方向与商业化 / 社区化尝试的蓝图。

---

## 一、 当前核心优势 (Current Value Prop)

1. **信息差抹平**：官方系统只能通过人为登录刷新查看，本系统通过 3 分钟级高频监控，将「被动查找」变为「主动推送」。
2. **免密无感登录**：摒弃复杂的密码管理，使用 Email Token 魔法链接直达个人门户。
3. **退课捡漏 (Snipe) 机制**：针对"已满"的热门课程（如射击体验、热门讲座），一旦有人退选空出名额，系统直接最高优先级通知订阅者。
4. **柔性骚扰控制**：根据开课时间实现 🔴即时 / 🟡3小时 / 🟢12小时 / 🔵每日汇总 的漏斗式推送策略，避免「通知风暴」。
5. **苹果级 UI 门户**：为无聊的教务系统套上了一层极客且优雅的外衣，极大提升用户使用愉悦度。

---

## 二、 近期产品迭代计划 (Short-term Roadmap)

### 1. 推送历史追溯
- **痛点**：用户收到邮件后，有时想在门户里查看过去 24 小时到底推送了哪些消息，有没有遗漏。
- **方案**：在门户（`portal.html`）新增一个「通知中心」Tab，展示用户最近收到的推送流（按时间线排列），区分「新发」与「退课补录」。

### 2. 管理端用户增长面板 (Admin Dashboard)
- **痛点**：目前 Admin 后台只有系统运行状态和课程概览。缺乏对 C 端用户的洞察。
- **方案**：新增 `Subscriber Management` (订阅者大盘)，一键查看注册总人数、活跃用户（最近访问门户时间）、热门订阅分类组合等，提供手动封禁 / 取消订阅的接口。

### 3. 多重通知渠道拓展 (Channel Expansion)
- 邮件由于反垃圾策略送达率可能偶有波动，对超级热门课程（剩2分钟开抢那种），邮件略慢。
- **方案**：
  - 接入 **Server酱 / 喵提醒 / pushplus** 等微信公众号推送接口。
  - 用户在个人的门户设置里，可以粘贴自己的 Webhook 或 Token。

## 三、 远期商业化与开源构想 (Long-term Vision)

### 1. 微信小程序生态 / 校园社区
- 将现有的 Web 门户打包为微信小程序。
- 建立一重「经验分享」功能：学长学姐可以给特定的常驻课程（如太极、某类讲座）打分（甚至标记**水课指数**、**抢手指数**）。
- 建立社交：一键将即将开抢的课程转发给微信好友/群，拼团组队签到。

### 2. 高级代抢服务 (Premium Auto-Enroll)
- 对于提供北航统一认证（SSO）账密的 Pro 用户，系统可以在检测到名额的第一秒，甚至不用发邮件，直接调用 `enroll.py` 内部逻辑代抢。
- **风控约束**：此项功能如果上线极其敏感，有可能被教务处封杀 IP，需引入动态代理池（IP Pool）或部署在具有大量校内节点的边缘网络。

### 3. 数据可视化战报 (Annual Stats)
- 用户年底可以查收一份「我的博雅年度报告」：“今年你一共参加了 X 场讲座，偏爱在【沙河校区】听课。你是全校前 5% 抢到《xxx》的幸运儿。” 这种功能利于朋友圈裂变传播。

### 4. 架构通用化 (SaaS for Universities)
- 项目底座剥离北航特定的 `check_in_method` 等字段，抽象为一套通用的「**教务抓取 + 降噪分发引擎**」。
- 其他高校可以通过编写特定的 Playwright Driver 插件接入这套生态。采用 Docker + Kubernetes (K8s) 水平扩展多个高校实例。
